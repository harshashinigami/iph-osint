## 1. What the Actual Product Is (per BEL PRD)

A **production-grade, AWS-hosted OSINT intelligence platform** with:
- 20+ live source connectors (Twitter, Facebook, Telegram, Reddit, Dark Web, etc.)
- Real-time Kafka pipeline processing millions of messages
- Neo4j enterprise knowledge graph with 15+ node types
- AI/ML models: IndicBERT sentiment, propaganda detection, financial fraud, deepfake detection
- Keycloak RBAC with 2FA
- 6 analyst screens + admin dashboards
- E-SitRep auto-generation in PDF/Word/Excel/CSV
- AWS infrastructure: EC2 r6g.4xlarge/8xlarge, MongoDB Atlas, S3, CloudWatch
- Performance SLA: 10K posts/min ingestion, 1000 concurrent users, sub-second entity extraction

### What Would Be Real in Production

## 5. What We Took for Granted (Assumptions)

1. **Free tier is sufficient for demo** — Render.com free tier has cold starts, limited compute, no persistent storage across deploys
2. **SQLite locally, PostgreSQL on Render** — works for PoC, but MongoDB/Neo4j needed for production
3. **No authentication on API** beyond JWT — no Keycloak, no 2FA, no RBAC enforcement
4. **Synthetic data is representative** — seed data mimics real OSINT patterns but isn't validated against actual intelligence workflows
5. **Single-server deployment** — no horizontal scaling, no load balancing
6. **No dark web access** — requires Tor infrastructure, specialized crawlers
7. **English-only NLP** — no Indian language support in the PoC
8. **No real social media APIs** — Twitter/Facebook require paid API access + approvals

## 6. How Far from Real Application

| Dimension | PoC (Now) | Production (BEL) | Gap |
|-----------|-----------|-------------------|-----|
| Data sources | 2 (RSS + Telegram) | 20+ | ~90% remaining |
| NLP capability | Rule-based (VADER + regex) | ML-based (IndicBERT, spaCy, custom) | ~80% remaining |
| Graph | In-memory NetworkX | Neo4j Enterprise cluster | ~85% remaining |
| Infrastructure | Render free tier | AWS India (r6g instances) | Complete rebuild |
| Security | Basic JWT | Keycloak + 2FA + RBAC + audit | ~90% remaining |
| Performance | Single user, <1K posts/sec | 1000 users, 10K posts/min | ~95% remaining |
| UI/UX | 8 functional pages | 6 polished screens per wireframe | ~40% remaining |
| Testing | Manual + Playwright | Full ATP (25+ scenarios) | ~90% remaining |

**Overall: PoC demonstrates ~25-30% of the production system's functionality, but covers 100% of the module categories.**

The PoC proves we can build the full pipeline. The delta is scaling, hardening, and replacing rule-based components with ML models.
